    Using the IBM Event Processing no-code editor to configure the solution

    As a result of your work so far, the marketing department now has a fully configured, self-service catalog of event stream data available within IBM Event Endpoint Management (EEM).

    At this stage, work will transition from the integration team to the domain of the marketing team. Focus Corporation's marketing team will utilize IBM Event Processing (EP) to correlate any newly-created customer accounts with first-time purchase orders that total over $100 USD in value. Furthermore, these "first time customer with greater than $100 transaction" correlation will need to be identified within a 24-hour window of when they occur.

    If these conditions can be met, the marketing team will be able to create an actionable plan for putting high-value promotional offers in front of these first-time customers in a timely manner — which should translate to more sales revenue through Focus' digital storefront.

    1. With your web browser, access the IBM Event Processing dashboard. Dismiss any welcome screens or tutorials that are prompted after first logging in.

    EVENT PROCESSING FLOWS

    An event processing flow ingests a stream of events, analyzes that input, and takes automatic actions (defined by the user) in response to those conditions.

    A flow begins with one or multiple event sources which represent inbound events. For the purposes of this demonstration, you will leverage the ORDERS event stream that was defined in the previous module. You will do so using EP's intuitive, no-code authoring canvas.

    1. In the bottom-right corner of the page, click the blue Create to begin defining a new flow.

    1. For Name, specify NewCustomerLargeOrder.

      When satisfied, click Create.

    1. Before the inbound events can be captured, connectivity details for the event source node (ORDERS) must be configured within the authoring canvas.

      • Wait until the page has refreshed to display the no-code environment. Along the left side are tools and actions that can be dragged into the authoring canvas on the right. Within the canvas, you will see an Event source node with default name of source_1.

      • Click the source_1 node and then click the Edit (pencil) icon in the top-right corner of the tile to begin modifying the node.

    1. The Configure event source wizard will load within your browser.

      • Under the Add an event source of pick a recent one header, select the Add event source tile.

      • When satisfied, click Next.

    1. Switch back to the Event Endpoint Management tab and return to the home dashboard.

      • Scroll down the landing page until you reach the Access Information section.

      • Just below, within the Servers table, is the full address of the Event Endpoint Management catalog.

      • Copy the full address (ademo-event-gw-ibm...) to your notepad for reference later.

    1. The first tab, Details, needs additional information to configure the node.

      • Scroll down to the Node name field and enter Orders

      • Further down, under the Connect to Kafka cluster header, paste the full address copied in Step 6 into the Server field.

      • When satisfied, click Next.

    1. The event source must be configured to allow certificates used by the event stream.

      • Accept certificates (by clicking the toggle button as shown in the screenshot below).

      • When satisfied, click Next.

    1. The second tab, Access credentials, requires authentication details for the EEM event source.

      • For Username and Password, supply the credentials for the ORDERS topic (Step 32 of Module 2).

      • When satisfied, click Next.

    1. The third tab, Topic selection, will prompt the user on which topic(s) to ingest from the event source.

      • Available topics will already be displayed within the configuration tool. Under the Topic selection header, click the button next to the CUSTOMERS topic to select it.

      • When satisfied, click Next.

    1. The fourth tab, Define event structure, requires a data schema or sample message in order to understand the expected structure of events ingested from the event source.

      • Under the No defined structure header, click the blue Upload a schema or sample message button to pull open an editor.

      • Users can input schemas in either Avro or JSON data formats. Click the JSON tab.

      • Copy and paste the following JSON into the sample message field. Note that this is the same sample schema you supplied when configuring the event source within EEM in Module 2.

    {
  "customerid": "acb3eb65-98a1-45c2-84d4-f5df157862b4",
  "customername": "Emilio Quitzon",
  "registered": "2023-10-24 19:20:35.638"
}

    When satisfied, click Done to proceed.

    1. You will have a final opportunity to review and confirm the event source configuration. At this time, the event structure should reflect the expected properties— customerid, customername, and registered fields —for the ORDERS event stream topic.

      When ready, click Configure to finalize the configuration.

    1. With the event source fully configured within EP, you can now begin tailoring the event processing flow to filter on orders where the value exceeds $100 USD.

      • The web browser will have loaded back to the authoring canvas, with the source_1 default node now properly configured as Orders.

      • Scroll down within the list of nodes and actions on the left-side interface until you locate the Filter node.

      • Click and hold the Filter node, then drag it to the authoring canvas on the right. Place it to the right side of the Orders node.

    1. The output terminal for Orders needs to be connected as input to the filter_1 node.

      • With your mouse pointer, hover over the Orders node and take note of the grey dot (labelled Output Port) at the right edge of the tile.

      • Click and hold the Output Port (right-most) of Orders and drag it to the Input Port (left-most edge) of filter_1, then release your hold to complete the circuit.

    1. With the Orders event source node connected to the filter_1 operation node, you must utilize the Filter node's expression builder to remove orders that are less than $100 USD in value. This will allow the marketing team's application to target only first-time customers with spending patterns of $100 USD or more.

      Hover over the filter_1 node with your mouse pointer and click the Edit (pencil) icon.

    1. Within the Details tab, specify the value of Node name as FilterLargerOrders.

      When satisfied, click Next.

    1. The Define filter tab will load next. To the right of the Filter expression field, click the Assistant drop-down menu to expose additional options.

    1. Event Processing's expression builder will guide you in defining the filter.

      • Property to filter on: price

      • Condition: Is greater than

      • Value: 100

      • When satisfied, click the Add to expression button.

    The expression builder will automatically translate your statements into a syntactically-correct filter statement: 'price' > 100

    Click the Configure button at the bottom of the panel to finalize the filter configuration.

    The marketing team needs to "JOIN" the filtered Order stream data to records of New Customers in order to correctly identify first-time purchasers that placed an order for over $100 USD in the last 24 hours. You shall use EP's authoring canvas to carry out the necessary JOIN operation.

    1. Back on the canvas for NewCustomerLargerOrders, scroll down along the left-hand interface until you locate the Interval Join node.

      Drag and drop this node at the far right-hand edge of the authoring canvas.

    1. With your cursor, hover over the New Customers output terminal and locate the Output Port (grey) button.

      • Click and hold the New Customers Output Port edge and drag this to the Input Port edge of the newly-created intervalJoin_1 node.

      • Release your cursor to finalize the connection.

    1. Repeat the process, this time connecting the FilterLargeOrders node's Output Port to the intervalJoin_1 node's Input Port.

      At this stage, the input of intervalJoin_1 should be connected to the outputs of both the FilterLargerOrders and New Customers nodes.

    1. The JOIN node must now be configured to correlate events based on a shared customerid field within the two event streams (New Customers and pre-filtered Orders).

      With your cursor, hover over the intervalJoin_1 node and click the Edit (pencil) icon.

    1. Within the Details tab, set Node name equal to DetectNewCustomerLargeOrder.

      When ready, click Next.

    1. Within the Join conditions tab, activate the expression builder by click the Assistant drop-down menu. Set the following properties:

      • Specify property from 'New Customers': customerid

      • Specify property from 'FilterLargerOrders': customerid

      • When ready, click Add to expression

    The syntactically-correct JOIN expression will be expressed under the Define events field:

    'New Customers'.'customerid' = 'FilterLargerOrders'.'customerid'

    After you have reviewed the JOIN condition, click Next.

    TRIGGERING EVENTS AND DETECTED EVENTS

    Two concepts are important to understand at this time: Triggering events and Detected events.

    • Since the New Customer sign-up event must logically occur before a purchase, it is considered a triggering event

    • Therefore, when a signal for an Order purchase of over $100 (the detected event) is received, the JOIN logic condition will be triggered for the two event streams

    1. Under the Time window condition, you can define the time interval "window" where detected events are considered viable for a promotional offer from the marketing team. To meet the criteria, a purchase of over $100 USD must be made by a first-time customer within a 24 hour window of creating an account.

      • Look for the Event to detect field and select the FilterLargerOrders (event_time) option from the drop-down menu.

      • A Preview visualization of the time window interval is rendered within the panel. For now, with no adjustments made, this will appear as a vertical bar at 0 (X-axis).

      • Scroll down further to reveal additional attributes to edit.

      • Event to set the time window: New Customers (event-time)

      • Offset from event to start the time window: 0 HOUR(S)

      • Offset from event to end the time window: 24 HOUR(S)

      • When ready, click Next

    The JOIN node's output going forward will be a combination of fields from both the filtered Orders and New Customers events streams.

    Before finalizing the interval join, you need to clean up the output so that duplicate fields (like customerid and event_time) are not included from the JOIN operation. You have the option of renaming or removing the duplicate fields — but for the sake of this demonstration, you will be removing the fields.

    1. Within the Output properties tab, locate the customerid field. There are two (duplicate) copies of the same field at this time. Your objective will be to delete ONE of the two duplicate records. Preserve one of the two.

      • Click the round — sign to the left of the first customerid row to remove the duplicate.

      • The result will be only one copy of the customerid field in the table. The red text (Must be unique) underlining the field should now have disappeared.

    1. Repeat the procedure for the event_time field, which is also duplicated by the JOIN operation.

      • Delete one of the two event_time fields

      • With all duplicates now removed from the table, the Configure button will now be highlighted in blue and selectable.

      • Click the Configure button to finalize the interval join configuration.
